## 2. 상세 수정 내역

### 2-1. SQL 마이그레이션 — 026_security_hotfix.sql (신규)

**이슈**: 보안 리뷰에서 3개 심각(Critical) 이슈 발견

| 수정 | 설명 | 심각도 |
|------|------|--------|
| anon INSERT 차단 | `cafe_requests` 테이블에서 비인증 사용자의 INSERT를 REVOKE | Critical |
| admin RPC 보호 | `get_admin_photo_spots` 함수에 관리자 UUID 체크 추가 | Critical |
| get_spots_by_ids 재작성 | anon 접근 제거, photo_url 반환 추가, 배열 500개 제한 | High |

```sql
-- 핵심 변경
REVOKE INSERT ON cafe_requests FROM anon;
-- admin-only RPC에 UUID 체크 추가
IF auth.uid() != 'da2a8b72-...' THEN RAISE EXCEPTION ...
-- array 크기 500 제한으로 DoS 방지
IF array_length(p_ids, 1) > 500 THEN RAISE EXCEPTION ...
```

**적용 방법**: `supabase db push` 또는 Supabase 대시보드 SQL 에디터에서 실행

### 2-2. cleanup_dev_data.sql → dev-scripts/ 이동

**이슈**: DELETE * / TRUNCATE가 포함된 스크립트가 production migration으로 배포될 위험

**수정**: `supabase/migrations/020_cleanup_dev_data.sql` → `supabase/dev-scripts/cleanup_dev_data.sql`로 이동하여 `supabase db push` 시 자동 실행되지 않도록 격리

### 2-4. settings_screen.dart (1건 수정)

**이슈**: `_deletePhoto`가 확인 없이 즉시 삭제, try-catch 없음

**수정**:
- 삭제 전 확인 다이얼로그 추가 (`AlertDialog`)
- `deleteSpotPhoto()` 호출로 변경 (Storage 파일도 함께 삭제)
- `try-catch`로 에러 처리 및 SnackBar 피드백

---

### 2-5. spot_detail_screen.dart (2건 수정)

**이슈 1**: `_spotPhotoProvider`의 family key가 `SpotModel` 전체 객체
- SpotModel에 == 없으면 매 build마다 새 인스턴스로 판단 → 불필요한 API 재호출
- **수정**: family key를 `String` (googlePlaceId)으로 변경, Supabase URL은 직접 사용

**이슈 2**: 이미지 로드 시 깜빡임 (팝인 현상)
- **수정**: `Image.network`에 `frameBuilder` 추가 — 300ms fade-in 전환

---

### 2-6. spots_repository.dart (3건 수정)

| 수정 | 설명 | 리뷰 출처 |
|------|------|-----------|
| `deleteSpotPhoto()` 신규 | Storage 파일 삭제 후 DB URL null 처리 (기존: DB만 null → 고아 파일) | 보안 |
| magic bytes MIME 검증 | 확장자 기반 → 파일 헤더(JPEG/PNG/WebP) 기반 검증으로 변경 | 보안 |
| URL whitelist 검증 | `updateSpotPhoto`에 도메인 화이트리스트 추가 (supabase.co, googleapis.com 등) | 보안 |

```dart
// magic bytes 검증 예시
static String? _validateImageMagicBytes(Uint8List bytes) {
  // JPEG: FF D8 FF
  if (bytes[0] == 0xFF && bytes[1] == 0xD8 ...) return 'image/jpeg';
  // PNG: 89 50 4E 47
  // WebP: RIFF....WEBP
}
```

### 2-7. spot_model.dart (1건 수정)

**이슈**: SpotModel에 `==` / `hashCode`가 없어서:
- Riverpod family provider가 동일 스팟도 다른 인스턴스로 판단
- Set/Map 연산에서 중복 허용

**수정**: `id` 기반 `operator ==` 및 `hashCode` 오버라이드 추가

---

### 2-8. places_service.dart (1건 수정)

**이슈**: `'pagetoken': ?pageToken` — Dart 3.4+ 전용 null-aware element 문법

**수정**: `if (pageToken != null) 'pagetoken': pageToken` — Dart 3.0+ 호환

## 4. 적용 순서 권장

1. **SQL 먼저 적용**: `026_security_hotfix.sql` → Supabase에 배포
2. **Dart 코드 배포**: 모든 .dart 파일 변경사항을 Git에 커밋
3. **테스트**:
   - 사진 업로드 → magic bytes 검증 동작 확인
   - 사진 삭제 → 확인 다이얼로그 + Storage 파일 제거 확인
   - 카페 상세 → 이미지 fade-in 전환 확인
   - 내 지도 → mounted 체크로 크래시 없음 확인
   - pageToken 페이지네이션 → 전체 카페 목록 정상 로드 확인
